tomcat的基本知识

发表于2023-01-24|更新于2023-02-01
什么是 Tomcat

Tomcat 是一种 web 服务器，用于部署运行 servlet 和 jsp 文件，响应来自浏览器的访问。

Tomcat 的目录结构如下

在 bin 目录下使用 ./startup.sh 或 ./catalina.sh run 即可启动tomcat，前者将在后台启动，控制台无日志信息，而后者将日志信息输出在控制台。使用 shutdown.sh 即可关闭（前台启动 ctrl+c 强制关闭）。

tomcat 默认监听 8080 端口，可以在 conf/server.xml 文件中修改 <Connector> 标签的 port 属性来指定监听端口

部署 web 工程
1. webapps 下部署

需要部署的 web 工程文件都需要放在 webapps 文件夹下。

通过 https://ip:port 将直接访问 ROOT 目录，默认请求 index.html/index.jsp 文件。

若指定访问工程名 https://ip:port/project 将访问 webapps 目录下的 project 路径，默认请求 index.html/index.jsp 文件。

若指定文件名 https://ip:port/project/test.html 将请求目标文件。

打包成 war 包后放入 webapps 文件夹，在启动 tomcat 时将自动解压 war 包 。

2. xml 配置部署

上一种方法必须把工程文件放在 webapps 文件夹下，若想访问其他路径的 web 工程，则需要使用 xml 文件配置。

有两种配置方法：

  1. 使用 conf/server.xml 文件配置
  2. 在 conf/Catalina/localhost/ 目录下新建 project.xml 文件配置

通常我们不使用第一种配置方法， 因为 server.xml 文件在 tomcat 启动后便不再读取，因此使用该文件配置需要重启 tomcat。

使用第二种配置方法可以分项目管理配置。

在 conf/Catalina/localhost/ 下新建 project.xml 文件。

此处 project 应为要部署的工程名，即配置完成后浏览器访问地址为 https://ip:port/project

project.xml 文件内容如下

文件中的 Context 标签用于指定项目上下文，其 path 属性指定访问路径，此处指定为 https://ip:port/project

docBase 属性指定工程实际存储的路径

path 属性可以省略不写，因为 tomcat 会直接将 project.xml 的文件名作为访问路径而忽视掉 path 属性

因此推荐直接将 xml 文件名作为访问路径，省略 path
