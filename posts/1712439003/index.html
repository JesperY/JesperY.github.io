No.217 存在重复元素【数组】

题目描述

给你一个整数数组 nums 。如果任一值在数组中出现 至少两次 ，返回 true ；如果数组中每个元素互不相同，返回 false 。

提示：

  • 1 <= nums.length <= 105

  • -109 <= nums[i] <= 109

解题思路
  • 判断一个数组中是否存在重复的元素，最暴力的思路就是嵌套循环遍历，对于外层遍历的每一个元素，通过内层遍历判断数组中是否还存在相同的元素，由于每个元素都被其之前的元素检查过一次，因此内层遍历只需要从外层当前位置 + 1 开始即可。由于使用了嵌套循环，该算法最坏时间复杂度为 1+2+3+….+n，即 O(n2)，空间复杂度为 O(1)。
  • 已知元素为整数，是可排序元素，考虑其排序结果是否有助于解决问题，该题中相同的元素在排序后一定相邻，因此通过检查是否存在相邻且相等的元素即可判断是否存在相同元素，该操作只需要一次遍历，加上排序所消耗的时间，最坏时间复杂度为 O(nlogn)，考虑排序所消耗的空间，空间复杂度为 O(logn)。
  • 对于不重复集合，很自然的想到 Set，将数组中的元素 add 到 Set 中，如果添加失败则说明存在重复元素。不考虑 Set 内部实现，时间复杂度和空间复杂度都为 O(n)。
  • 不重复集合也可以使用 Map 的 key，方法同上。

No.53 最大子数组和

题目描述

给你一个整数数组 nums ，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

子数组 是数组中的一个连续部分。

提示：

  • 1 <= nums.length <= 105
  • -104 <= nums[i] <= 104
解题思路

求连续最大子数组的和，典型的动态规划问题，假设最大子数组以第 i 个元素结尾，设其和为 f(i)，则我们只要求出所有的 f(i)，取其中最大值即可。

如何求 f(i) 呢？根据其连续特性，第 i 个元素构成的最大子数组有两个选择，即 f(i) = max{ f(i-1) + nums[i], nums[i] }。

在我们顺序求解 f(i) 时，可以同时进行取最大值的操作而不需要将每个 f(i) 都保存下来，这样其时间复杂度为 O(n)，空间复杂度为 O(1)。

No.1 两数之和

题目描述

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 target 的那 两个 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。

你可以按任意顺序返回答案。

提示：

2 <= nums.length <= 104

-109 <= nums[i] <= 109

-109 <= target <= 109

只会存在一个有效答案

解题思路
  • 一眼暴力思路，对于每一个元素，遍历其后的元素是否存在 i + j = target。如果存在则返回 { i , j }。该题目已经假定有且仅有一个有效答案，因此 if 外的 return 语句随意返回即可。此方法最坏时间复杂度 O(n2)。
  • 由于返回结果是数组下标，因此不能对数组进行排序，题目也没有说明数组有序，因此不能使用二分查找来优化内层查找。
  • 很显然外层遍历不可避免，因此重点仍在内层查找优化上，除了二分查找，还有一种更加快速的方法，即哈希查找。通过遍历将每个元素存入哈希表中，存入前检查是否存在 target - nums[i]，若不存在则存入，若存在则查找成功。哈希查找时间复杂度为 O(1)，外层遍历为 O(n)。因此该算法最坏时间复杂度为 O(n)，空间复杂度为 O(n)，以空间换时间。

No.88 合并两个有序数组

题目描述

给你两个按 非递减顺序 排列的整数数组 nums1 和 nums2，另有两个整数 m 和 n ，分别表示 nums1 和 nums2 中的元素数目。

请你 合并 nums2 到 nums1 中，使合并后的数组同样按 非递减顺序 排列。

注意：最终，合并后数组不应由函数返回，而是存储在数组 nums1 中。为了应对这种情况，nums1 的初始长度为 m + n，其中前 m 个元素表示应合并的元素，后 n 个元素为 0 ，应忽略。nums2 的长度为 n 。

提示：

nums1.length == m + n

nums2.length == n

0 <= m, n <= 200

1 <= m + n <= 200

-109 <= nums1[i], nums2[j] <= 109

解题思路

两数组合并问题，而且还是有序数组的合并，非常自然地想到 O(m+n) 的双索引顺序合并。但是题目要求将 nums2 合并到 nums1 中，又涉及到 nums1 中元素的移动。元素移动的时间复杂度可以认为是 O(n2)，因此应尽可能避免，由题意知两数组长度不大，采用空间换时间的方法，将 nums1 的元素复制下来，然后进行合并。这种方法的最坏时间复杂度为 O(m+n)，空间复杂度为 O(m)。

另一种思路，由于 nums1 已经预留了 nums2 的位置，因此考虑从后往前，以非递增的条件逆序插入，从而保证顺序非递减的要求，此种方法需要保证 nums1 中未被使用的元素不被覆盖，显然由于预留了足够的位置，任何情况下都不会发生覆盖问题。最坏时间复杂度为 O(m+n)，空间复杂度为 O(1)。

第三种思路最为简单，直接将 nums2 复制到空余位置，然后对 nums1 进行排序，这种方法的时间复杂度依赖于排序算法，常用快排的时间复杂度为 O((m+n)log(m+N))。

题解

排序方法不再给出代码。

No.121 买卖股票的最佳时机

题目描述

给定一个数组 prices ，它的第 i 个元素 prices[i] 表示一支给定股票第 i 天的价格。

你只能选择 某一天 买入这只股票，并选择在 未来的某一个不同的日子 卖出该股票。设计一个算法来计算你所能获取的最大利润。

返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 0 。

提示：

  • 1 <= prices.length <= 105
  • 0 <= prices[i] <= 104
解题思路

动态规划问题，设第 i 天卖出股票的利润为 p(i)，则 p(i) 只和两个值有关，一个是 prices[i]，一个是 i 之前的最小值（因为只能先买入，然后才能卖出，所以买入价格一定在卖出价格之前）。设 min 是 i 之前的最小值，则 p(i) = prices[i] - min，而 min 的值可以随遍历同时求。

No.566 重塑数组

题目描述

在 MATLAB 中，有一个非常有用的函数 reshape ，它可以将一个 m x n 矩阵重塑为另一个大小不同（r x c）的新矩阵，但保留其原始数据。

给你一个由二维数组 mat 表示的 m x n 矩阵，以及两个正整数 r 和 c ，分别表示想要的重构的矩阵的行数和列数。

重构后的矩阵需要将原始矩阵的所有元素以相同的 行遍历顺序 填充。

如果具有给定参数的 reshape 操作是可行且合理的，则输出新的重塑矩阵；否则，输出原始矩阵

提示：

m == mat.length

n == mat[i].length

1 <= m, n <= 100

-1000 <= mat[i][j] <= 1000

1 <= r, c <= 300

解题思路

矩阵的 reshape 操作，毫无疑问一个新的空矩阵是必须要创建的，而对原矩阵的一次遍历也是必须得，重点在于转换操作。

因为我们并不需要原矩阵的行列信息，只是需要原矩阵的所有元素，从而映射到新矩阵，因此可以采用矩阵的一次遍历得到线性数组，再映射到新矩阵。
