No.278 第一个错误版本

题目描述

你是产品经理，目前正在带领一个团队开发新的产品。不幸的是，你的产品的最新版本没有通过质量检测。由于每个版本都是基于之前的版本开发的，所以错误的版本之后的所有版本都是错的。

假设你有 n 个版本 [1, 2, …, n]，你想找出导致之后所有版本出错的第一个错误的版本。

你可以通过调用 bool isBadVersion(version) 接口来判断版本号 version 是否在单元测试中出错。实现一个函数来查找第一个错误的版本。你应该尽量减少对调用 API 的次数。

提示：

  • 1 <= bad <= n <= 2^31 - 1
解题思路

本质是一个查找问题，要求尽可能的少的比较次数，很自然的想到二分查找。中值为正确版本，则目标值在右侧，中值为错误版本，则目标值在左侧，查找成功的条件为当前值为错误版本且前一个为正确版本。

如果在一次查找中比较 mid 和 mid - 1 的值，则一次查找要调用两次接口，为了避免这样一种情况—— mid 为目标值，但在不判断 mid - 1 的情况下直接令 right = mid - 1 会导致该目标值被排除掉。我们修改二分查找为 right = mid，从而避免边界情况，这样修改之后，如果 right 为目标值，则 right 不变，left 继续向 right 靠近直到 left = right。同理如果 right 始终没有取到目标值，也最终会通过二分查找步骤查询到目标值。

需要注意由于我们令 right = mid 而不是 right = mid - 1，因此循环结束条件不再是 left <= right 而是 left < right，显然如果 right 指向目标值则会导致 right 不断更新为同一个值进而导致死循环，而采用不等判断时返回值不能采用 mid，这是因为这种条件下 mid 无法取到 left = right 条件下所代表的中值，因此返回值只能是 right 或者 left。

根据提示得传入值在运算过程中有可能溢出，因此令 mid = left + (right - left) / 2。

No.35 搜索插入位置

题目描述

给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。如果目标值不存在于数组中，返回它将会被按顺序插入的位置。

请必须使用时间复杂度为 O(log n) 的算法。

提示:

1 <= nums.length <= 104

-104 <= nums[i] <= 104

nums 为 无重复元素 的 升序 排列数组

-104 <= target <= 104

解题思路

从题目要求不难看出需要使用二分查找。

根据题意知数组有序，要查找的目标元素为第一个大于等于 target 的元素，返回其下标。

No.283 移动零

题目描述

给定一个数组 nums，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。

请注意 ，必须在不复制数组的情况下原地对数组进行操作。

提示:

  • 1 <= nums.length <= 104
  • -231 <= nums[i] <= 231 - 1
解题思路

要求不能复制数组，就只能考虑数组内部移动。

本质是在保持元素顺序不变的情况下将所有的零移动到数组末尾，有点类似于排序算法，只是排序对象只针对零，又因为同时要保持非零元素顺序不变，这种排序过程不改变无关元素先后顺序的要求很像冒泡排序。

一种比较暴力的方法，遍历数组，对于每个零元素，查找其后的第一个非零元素，交换两者位置，直到数组遍历完毕。

整个交换过程还可以看做是所有非零元素的向前移动，因此可以考虑数组元素的平移，使用双指针遍历数组，一边遍历一边进行复制，即令 nums[j] = nums[i]，没有碰到零元素时，i == j，数组并没有发生任何改变，如果当前 i 遍历到零，则 j 停止一次自增且不进行赋值，此时 i = j+1，这样两个指针出现了偏移，零之后的元素相当于向前平移了一位，此过程要记录遍历到的零的个数，因为我们最后要把数组末尾填充为零。

No.167 两数之和 II - 输入有序数组

题目描述

给你一个下标从 1 开始的整数数组 numbers ，该数组已按 非递减顺序排列 ，请你从数组中找出满足相加之和等于目标数 target 的两个数。如果设这两个数分别是 numbers[index1] 和 numbers[index2] ，则 1 <= index1 < index2 <= numbers.length 。

以长度为 2 的整数数组 [index1, index2] 的形式返回这两个整数的下标 index1 和 index2。

你可以假设每个输入 只对应唯一的答案 ，而且你 不可以 重复使用相同的元素。

你所设计的解决方案必须只使用常量级的额外空间。

提示：

2 <= numbers.length <= 3 * 104

-1000 <= numbers[i] <= 1000

numbers 按 非递减顺序 排列

-1000 <= target <= 1000

仅存在一个有效答案

解题思路

两数之和， 由于题目要求 O(1) 的空间复杂度，因此 Map 的方法就不能用了，常规的暴力搜索时间复杂度太高。

题目中给出了额外的条件，即数组是有序的。在有序数组中同时进行两个元素的操作，自然考虑到双指针，寻找 nums[i] + nums[j] == target。

根据有序条件，考虑前后指针顺序搜索，即一个指针固定当前值，另一个指针顺序搜索后续值，当两者之和大于 target 时，前一个指针后移一位，继续同样的操作，直到找到目标值。即使通过排序可以排除掉所有 nums[i] + nums[j] > target 的情况，但其时间复杂度仍然是 O(n2)。

换一种思路，一个指针从最小值开始，一个指针从最大值开始，两元素之和只有三种情况，等于、大于或小于 target，等于即是结果，小于则要增大元素之和，因为最大值已经不能再增大，因此只能最小值指针增大，大于则要减小元素，因为最小值已经不能再减小，因此只能最大值减小。

No.3 无重复字符的最长子串

题目描述

给定一个字符串 s ，请你找出其中不含有重复字符的 最长子串 的长度。

提示：

  • 0 <= s.length <= 5 * 104
  • s 由英文字母、数字、符号和空格组成
解题思路

第一反应暴力遍历，对于每个起始字符搜寻其后的最长不重复子串。

乍一看有点像动态规划的题目，对于以索引 i 结尾的最长不重复子串命名为 f(i)，则如果 f(i-1) 中没有重复的字符，f(i) = f(i-1) + i，如果有重复字符，则需要将重复字符排除，很明显并不能将问题拆分为寻找 f(i-1) 和 f(i) 之间的关系，动态规划的思路也无法进行下去。

双指针的思路与暴力遍历类似，一个指针指向起始字符，另一个指针先后遍历并不断记录遍历过的字符，当遇见重复字符时，则需要将左指针后移至没有重复字符为止，这要求右指针遍历时记录每个字符的索引，重复时将左指针移动到重复字符的右侧。

上述双指针方法其实就是所谓的滑动窗口，我们只需要使用一个左指针维护窗口左边界，同时不断向右遍历扩展右边界，在出现重复字符时更新左边界，并不断记录窗口最长长度即可。

No.733 图像渲染

题目描述

有一幅以 m x n 的二维整数数组表示的图画 image ，其中 image[i][j] 表示该图画的像素值大小。

你也被给予三个整数 sr , sc 和 newColor 。你应该从像素 image[sr][sc] 开始对图像进行 上色填充 。

为了完成 上色工作 ，从初始像素开始，记录初始坐标的 上下左右四个方向上 像素值与初始坐标相同的相连像素点，接着再记录这四个方向上符合条件的像素点与他们对应 四个方向上 像素值与初始坐标相同的相连像素点，……，重复该过程。将所有有记录的像素点的颜色值改为 newColor 。

最后返回 经过上色渲染后的图像 。

解题思路

不难看出，实际上是对二维数组从给定坐标所开始的搜索算法。此处采用的是上下左右四个方向的搜索，相当于将每个坐标视为一个点，与上下左右四个方向上存在边从而构成一张图，这道题就转化为了图的遍历，无论是深度优先还是广度优先都可以满足题目要求，记录该题只是为了学习其二维数组的 DFS 与 BFS 思想。
