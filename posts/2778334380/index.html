算法-二分查找

发表于2023-03-11|更新于2023-05-25

二分查找是一个非常简单且基础的查找算法，可以在 O(logn) 的时间内完成查找。

我们看一个常规的二分查找模版。

这是一个非常简单的二分查找实现，对应最简单的查找场景，即我们要查找的目标值如果在数组中，返回其索引，如果循环结束仍没有找到，则返回 -1。

但实际上我们会根据一些细节来修改或者优化二分查找的实现。

1. 长度过大

当数组长度过长，可能导致 mid 溢出，因此我们可以修改 mid 的取值方法。

2. 循环条件

二分查找的循环条件有两种写法，即 while(left < right) 和 while(left <= right)，两者的结束条件显而易见，前者是 left = right，后者是 |left - right| = 1。那么如何区分两者写法呢？

  • while(left < right) 在循环结束时才能取到 left = right，即在循环体内无法取到 left = right 时的元素。但这种说法也并不觉得，需要和 left 与 right 的更新相结合，这个表述是在 left = mid + 1，right = mid - 1 的基础上提出的，其实并无意义，通常这种循环条件意在查找边界情况，而要对应特殊的边界更新，详细内容将在下一节讨论。
  • while(left <= right) 在循环体中可以查找到所有的元素，同样这是基于 left = mid + 1，right = mid - 1 所提出的。通常这种条件的循环体内对应三个分支，即大于，小于和相等。相等即是我们要查找的结果。循环体外表示查找失败，通常返回 -1。
3. 边界情况

很多时候我们并不是要找数组中的某个特定值，例如我们要查找第一个小于 target 或者大于等于 target，或者我们要找出插入 target 的位置。这种情况下我们要查找的元素位于边界，即所有大于等于 target 的元素的最左边的元素……诸如此类。在这种情况下我们要考虑 left 和 right 的更新方法。

请务必牢记，left 和 right 的更新意味着下一次要进行搜索的区间范围，而不仅仅是为了缩小而缩小，例如当我们查找第一个小于 target 的值时，如果 mid 的值小于 target，我们知道 target 可能在 mid 的右边，但必须注意，当前的 mid 仍然在下一次循环的搜寻范围内，因为你不能保证当前这个 mid 就不是 target。这就要求我们使用 left = mid 来更新，相对的我们可以保证 mid 大于 target 时，mid 一定不会是 target，因此 right = mid - 1。

只要牢记 left 和 right 是指定下一次查找的区间范围这个概念，根据题意灵活选择更新方法，就不会出问题。

刚才我们讨论到 while(left < right) 的循环条件，这就是和边界更新相关的写法。

不难发现，如果我们使用 while(left <= right)，很容易出现 left 和 right 相等时，left = mid 导致区间无法进一步缩小而循环也无法退出的情况，这就出现了死循环，因此对于边界问题，尽量避免使用相等判断，只要 left = right 时可以保证一定退出循环才能使用，但仍然不建议。

在 while(left < right)条件下，循环退出时一定是 left = right，因此处理结果不需要考虑是 left 还是 right，两者相等，通常这种情况下不使用 mid，因为 mid 无法取到 left = right 的值。

对于 mid 的更新除了 mid = (left + right) / 2 之外，还有 mid = (left + right + 1) / 2 的写法，这是为了在偶数区间时 mid 能够取到中间两个值的右值所做的优化。假如我们查找第一个小于 target 的值，即当 mid 值小于 target 时，left = mid，mid 值大于 target 时，right = mid - 1。假设给定数组 {1,5,7}，target 为 6。
