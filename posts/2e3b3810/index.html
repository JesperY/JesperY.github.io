AI-SearchProblem

发表于2023-08-03|更新于2023-08-09

1. Search Problem

搜索问题通常可以从三个方面入手：

  1. 状态空间（State Space）：包含了求解问题的过程中可能涉及的所有状态，分为世界状态和搜索状态：
    1. 世界状态（World State）：只该问题描述下所有可用的信息所组成的所有状态。
    2. 搜索状态（Search State）：只在搜索过程中使用的信息所组成的状态。
  2. 后继函数（Successor Function）：描述对于当前状态，如何进行下一步操作并转换状态和修改信息的函数。该函数通常需要 Action 和 Cost 信息。
    1. 操作（Action）：即在搜索过程中，后继函数可以执行的操作，执行该操作后将会转换到下一个状态。
    2. 代价（Cost）：不同的操作会有不同的代价，可能是对资源的占用，也可能是为了让函数尽可能不执行某一类操作而人为设置的代价。
  3. 开始状态（Start State）和目标测试（Goal Test）：搜索开始的位置以及如何判断是否已经达成目标。

有了上述定义之后我们就可以给出一个搜索问题的解决方案（Solution）：一组从开始状态到目标状态的操作。

2. State Graph And State Tree

2.1 State Graph

状态图是描述世界状态或者搜索状态的图。

通常我们会分析世界状态，但并不会去描述它，因为它包含的信息实在是太多了。

图中的每一个节点代表一个状态，边代表所执行的操作，边上的权重代表该操作需要的花费。

视不同的情况，图可能是有向图也可能是无向图

2.2 Search Tree

搜索树是搜索过程中构建的树，可以类比二叉搜索树。以起始状态作为根节点，每一个操作指向下一个状态。

由于递归搜索的过程总会涉及到重复计算，因此搜索树中存在很多重复的子树,这导致搜索树甚至比状态图还要大。

常用的树搜索算法不外乎 DFS、BFS。除此之外由于引入了代价（Cost），所以还有一种 UCS（Uniform Cost Search）代价一致搜索。这种算法每次选取代价最小的分支进行搜索。

3. Informed Search

启发式搜索，即有信息的搜索算法。上一节中提到的 DFS、BFS、UCS 都是无信息的搜索算法，它们对于目标状态（goal state）的位置并没有任何的认识，或者说没有任何的相关信息。而启发式搜索会根据已有的信息推断目标位置。

3.1 Heuristics

所谓的启发就是一个评估一个状态到目标状态的距离的函数。即 h(n)，这被称为启发函数。启发式搜索根据启发函数的结果来决定操作。

先来了解几个常用的符号：

  1. f(n)：评价函数，用于评价到 n 节点所需要的代价。
  2. g(n)：代价函数，即从初始状态到 n 节点实际需要的代价。
  3. h(n)：启发函数，评估 n 节点到目标节点所需要的代价。

举个例子，UCS 中就使用了 f(n) = g(n) 来作为评价函数，即仅仅以初始状态到达下一节点的代价作为评价标准，因此优先选择代价最小的路径。这种搜索方式不能保证找到最优路径。f(n) 通常是多种信息的组合，例如 f(n) = h(n) + g(n) 就是 A* 搜索的评价函数。

启发式搜索最重要的就是启发函数，是我们能否选择最优路径的关键之一。通常情况下启发函数给出的值会低于实际值，即 h(n) < (节点 x 到目标节点的实际最小代价)。这被称为可容的（Admissible）启发，即可容性。

h(x) 越接近真实值，则算法效果越好，设计启发函数也是启发式搜索的关键所在。

h(x) 又被称为将来代价（forward cost），g(n) 又被称为过去代价（backward cost）。

3.2 Greedy Best-First Search

贪婪算法（GBS）以 f(n) = h(n) 作为评价函数，优先选择到目标节点所需代价最小的节点。听起来这个算法似乎可以完美的选择出最优路径，其实不然，GBS 只考虑了下一节点 n 到目标节点的代价，但并没有考虑当前节点到下一节点 n 的代价，因此会出现”绕远路“的情况，只能达到局部最优效果。

3.3 A-star Search

A* 算法结合了 GBS 和 UCS 的特点，即 f(n) = h(n) + g(n)，这种策略可以实现全局最优，具体的证明此处就不再详谈。

需要注意，搜索算法在何时结束？

A* 算法会以 f(n) 为优先级构建优先队列，优先选取值最小的节点。当扩展一个新节点时，会计算它们的 f(n) 并将其入队，然后选取根据优先级选取下一节点进行扩展。因此当我们搜索到目标节点时，由于其 h(n) = 0 因此其可能是队列中优先级最高的节点，但是这并不绝对，队列中可能还存在优先级更高的节点，此时如果我们在入队时进行目标检查（goal test），则可能错过真正的最优路径。因此我们需要在出队时进行检查，因为出队所选取的节点一定是优先级最高的节点，可以保证最优路径。

3.4 Somethings

如何选取启发函数呢，根据可容性，启发函数通常低估代价，这又叫松弛问题，但是过于松弛的启发函数会导致搜索耗散。

启发函数的选择有很多，我们可以组合不同的启发函数，来取其精华。

对于 ha 和 hb，如果对于任意 n，ha 总是大于 hb，这被称为优势，我们总是可以选择 ha。

对于 ha 和 hc，如果并不存在优势，则可以使用 h(n) = max(ha, hc)。

在搜索树中我们可以发现很多节点被重复搜索了，直接上这些子树是完全相同的，为了避免重复搜索，我们可以构建一个闭集（closed set），用于存储已经被搜索过的节点，从而跳过重复搜索。

3.5 Admissible & Consistency

可容性与一致性。

3.5.1 Admissible

可容性：0 <= h(n) <= h*(n)，即启发函数进行低估，一旦估计代价超过实际代价，搜索将失去意义。

3.5.2 Consistency

一致性：h(A) - h(C) <= Cost(A to C)

h(A) - h(C) 表示从 A 到目标的估计代价减去从 C 到目标的估计代价，其值表示从 A 到 C 的估计代价。

因此一致性是指 A 到 C 的 估计代价小于 A 到 C 的实际代价。
