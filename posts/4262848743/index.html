Python 爬虫之 BeautifulSoup 库

发表于2023-05-23|更新于2023-05-25

创建 BeautifulSoup 对象

导入 BeautifulSoup 包后，可以使用 html 格式的字符串或者 html 文件来创建一个 BeautifulSoup 对象。

推荐使用 lxml 作为解析器，如不指定则默认使用 Python 原生 html.parser 解析。

BeautifulSoup 并不参与文档解析。

BeautifulSoup 采用 unicode 编码来处理文档，处理时会自动进行编码转换，然后将文档转换为一个树形结构，每个节点都是 Python 对象。

所有对象都可以分为以下四种：

  1. Tag：表示 xml 或 html 原生文档中的 tag，该对象在遍历和搜索文档树中起主要作用。

    Tag 有两个最重要的属性，即 Name 和 Attribute。

    每个 tag 都有自己的 name，通过 tag.name 可以获取 tag 的名称，例如 p、div、a 等，修改 name 将直接改变 tag 类型。

    tag 可能有零个或者多个 attribute，通过 tag['attr_name'] 的形式可以操作对应的 attr 值，或者 tag.arrts 获取一个包含所有 attr 的字典，该字典可以像 python 字典一样被操作，操作结果也会反映在文档树中。

    某些 attr 例如 class 会有多个值，被称为多值属性。html 规范中规定的多值属性会被返回为 list 类型，否则返回为字符串。

    xml 规范中不存在多值属性，因此全部返回为字符串。

  2. NavigableString：表示被 tag 包含的字符串。

    html 中的 tag 可能包含字符串内容用于显示。使用 tag.string 就可以获取当前 tag 所包含的字符串，该字符串不能被修改，但可以使用 tag.string.replace_with()方法替换，支持文档树的大部分属性，但因为其不能包含子内容，因此该部分相关属性不能使用。

    如果需要在文档树之外使用可遍历字符串，则需要使用 unicode() 方法将其转换为普通字符串，否则其将作为一种自定义对象被使用。

  3. BeautifulSoup：一个文档的全部内容。

    可以近似看作一个 tag，支持文档树大部分操作。但由于并不是真正的 html 或 xml 的 tag，因此并不具有 attribute，其 name 属性值为 [document]，仅为了方便使用。

  4. Comment：注释，一种特殊的 NavigableString。

    本质也是一个可遍历字符串，但会使用特殊格式输出。

遍历文档树

对文档树的遍历就是对节点的遍历。本质就是树形结构的操作。

tag.name 取标签

使用 tag.name 可以获取到当前 tag 下第一个匹配 name 的 tag 对象。

该方法可以嵌套，即 soup.div.p.div.a 形式。

.contents 和 .children

使用 tag.contents 可以获取 tag 的子节点列表。

soup 对象包含 html 标签。

使用 tag.children 则返回一个子节点的迭代器。

.descendants

上面两个属性只能获取直接子节点，该属性可以获取所有子孙节点。

该属性返回值是一个 generator 对象。

使用 yeild 关键字可返回 generator 对象，该对象不会一次生成整个列表，而是每次循环返回一次。

.string

当 tag 只有一个 NavigableString 类型子节点时，可以使用该属性获得。

当 tag 只有一个子节点时，则使用该属性相当于对子节点使用该属性。

如果有多个子节点，则返回 None。

.strings 和 .stripped_strings

如果包含多个字符串，则使用 .strings 来循环获取。该属性同样返回一个 generator 对象。

使用 .stripped_strings 可以去除空格和空行条目。

.parent 和 .parents

.parent 可以获取父节点。

html 的父节点是 BeautifulSoup 对象，BeautifulSoup 对象的父节点是 None

.parents 可以获取所有父辈节点。

.next_sibling 和 .previous_sibling

查询兄弟节点。

兄弟节点必须拥有相同的父节点。

需要注意 html 文档中的各个标签之间存在着空白字符，这些也被视为兄弟节点。

.next_siblings 和 .previous_siblings

返回对多个兄弟节点的 generator 对象。

.next_element 和 .previous_element

指向下一个被解析的对象，其结果和兄弟节点并不总是相同。

解析的过程会逐渐进入 tag 内部，因此下一个被解析节点通常是子节点。

.next_elements 和 .previous_elements

返回 generator 对象。

搜索文档树

搜索主要使用两个函数，但搜索方法并不仅仅局限于这两个函数。

过滤器

过滤器是一些可以用于搜索函数的参数的类型。主要有以下几种：

  1. 字符串：传入一个字符串参数，则会搜索完整匹配字符串的内容。
  2. 正则表达式：传入一个正则对象，则会根据正则进行匹配。
  3. 列表：列表中可以有多个过滤器，返回匹配任一元素的内容。
  4. True：返回所有值。
  5. 方法：传入一个方法，该方法接受一个 tag，根据返回的布尔值确定是否被找到。
find() 和 find_all()

find_all(name, attrs, recursive, string, **kwargs)

该方法搜索当前 tag 下的所有子 tag，并判断是否符合传入的过滤器条件。

该方法的不同参数用于过滤不同的内容。

  1. name：查找所有 name 属性符合传入过滤器的内容。

  2. attrs：接受一个字典类型，使用字符串类型键值对来指定过滤的属性键值对。

    此参数通常用于过滤某些不能使用 **kwargs 指定的属性

  3. kwargs：当指定一个参数名不是形参列表中的时，会将其参数名作为属性名，搜索每个 tag 对应的属性值是否符合参数值。

  4. string：搜索字符串内容，接受过滤器类型。默认搜索字符串内容，和其他参数混合使用时可用于过滤 .string 属性。

  5. limit：整型，限制返回的结果数量。

  6. recursive：默认为 True，搜索所有子孙节点，设置为 False 时只搜索直接子节点。

BeautifulSoup 4.1.1 版本开始可以使用 class_ 形参指定匹配 tag 的 class 属性。

BeautifulSoup 实际上提供了很多 DOM 树的搜索方法，参数基本相同，但常用的只有这两个。

可以直接使用 tag(name, attrs, ....) 的形式调用。等同于 tag.find_all(name, attrs, ....)。

find() 方法直接返回结果，且仅返回一个。

find_all() 方法没找到时返回空列表，find() 返回 None。

find_parents() 和 find_parent()

搜索父辈节点。

find_next_siblings() 和 find_next_sibling()

搜索后续的兄弟节点。

find_previous_siblings() 和 find_previous_sibling()

搜索前面的兄弟节点。

find_all_next() 和 find_next()

对当前 tag 之后的 tag 和字符串进行搜索。

find_all_previous() 和 find_previous()

对当前 tag 之前的 tag 和字符串进行搜索。

CSS 选择器

使用 Tag 的 select() 方法传入一个字符串参数，可以使用 CSS选择器语法进行搜索。

使用 select_one() 可以返回第一个结果。

选择器语法请查看[CSS选择器语法]
