VS2022 打开 .net framework 4.0 工程提示 target framework not supported

发表于2023-04-28|更新于2023-04-28

想要学习一下一个 GitHub 上的 C# 项目的源码，项目比较老，使用的还是 .Net Framework 4.0 ，打开时提示 Target Framework Not Supported 错误，并且资源管理界面中显示 project not loaded。

在项目上邮件选择 reload 后仍然提示 Not Supported，并且给出的修复选择中 Update 选项无效。

无法得知具体的错误原因，也找不到 .Net framework 4.0 SDK 的下载渠道，也尝试过添加 VS2017 目标包，均无效。

在以上操作的基础上，最终通过修改项目配置文件成功过打开，下面给出具体步骤。

  1. 首先是安装 vs 2017 目标包，我并不知道这步是否有作用，但还是给出下载链接，该页面可能需要登录微软账号并加入 Dev Essential 计划才能免费下载，否则就需要花钱订阅。我使用的是 Build Tools for Visual Studio 2017 (version 15.0) 目标包。

  2. 修改 App.config 文件，将下述标签中的版本改为错误提示中推荐 Update 的版本，我没有尝试改为最新或者其他版本，请自行斟酌。

  3. 修改 .csproj 文件，方法同上。需要注意，改标签可能存在多个，建议全部修改。

修改完成后 reload 工程，即可成功打开。
