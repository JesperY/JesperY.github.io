Python 爬虫之 Selenium 库

发表于2023-05-25|更新于2023-08-09

1. 获取浏览器

获取浏览器句柄后我们就可以调用其 api 来执行各种方法，模拟我们对浏览器的操作。

get 方法会等待网页打开后返回，但不会等待 ajax，因此对于使用 ajax 的网页建议睡眠一段时间或者使用 wait 方法。

可以通过 driver 对象的属性和方法获取一些存储在浏览器中的数据

2. 访问 url

通过 get 方法访问一个网页，返回的信息会存储在 driver 对象中，driver 就相当于一个浏览器，存储了所有返回的信息。

3. 获取网页中的各个元素

在最新的 selenium 4.9 中，使用 find_element 系列的方法更换了新的 api。

该方法返回一个 element 对象，可以对其进行操作。如果存在多个匹配，则只返回第一个。

使用 find_elements 系列方法可以返回 element 对象的列表。

该方法系列有以下方法（注意使用小写）：

  • FIND_ELEMENT(By.ID, “”)
  • FIND_ELEMENT(By.NAME, “”)
  • FIND_ELEMENT(By.XPATH, “”)
  • FIND_ELEMENT(By.LINK_TEXT, “”)
  • FIND_ELEMENT(By.PARTIAL_LINK_TEXT, “”)
  • FIND_ELEMENT(By.TAG_NAME, “”)
  • FIND_ELEMENT(By.CLASS_NAME, “”)
  • FIND_ELEMENT(By.CSS_SELECTOR, “”)
  • FIND_ELEMENTS(By.ID, “”)
  • FIND_ELEMENTS(By.NAME, “”)
  • FIND_ELEMENTS(By.XPATH, “”)
  • FIND_ELEMENTS(By.LINK_TEXT, “”)
  • FIND_ELEMENTS(By.PARTIAL_LINK_TEXT, “”)
  • FIND_ELEMENTS(By.TAG_NAME, “”)
  • FIND_ELEMENTS(By.CLASS_NAME, “”)
  • FIND_ELEMENTS(By.CSS_SELECTOR, “”)

这些方法没有找到时会抛出异常。

4. 模拟键盘操作

send_keys 接受一个 *value 参数，这是一个可变参数，说明可以传入多个键以实现组合键效果。

from selenium.webdriver.common.keys import Keys 可以导入功能键：

  • Keys.ENTER

  • Keys.BACK_SPACE

  • Keys.SPACE

  • Keys.TAB

  • Keys.ESCAPE

  • Keys.F系列功能键

全部枚举值请查看文档

5. Select 元素

对于下拉列表元素，虽然我们可以通过遍历其所有 option 标签来选中，但这是比较复杂。

selenium 提供了处理下拉列表的类型，使用 element 对象创建一个 select 对象，使用其方法就可以选择其中的 option。

select 对象提供了一些属性用于获得一些有用的信息。

6. Action Chains

ActionChains 对象，顾名思义，一个动作链，其设置的动作会被保存，但不会马上执行，而是会在调用其 perform 方法后连续执行所有动作。

使用 driver 创建一个 ActionChains 对象。

7. 切换窗口

window 和 frame 均属于窗口的范围。

通过遍历句柄，判断是否是所需要切换的窗口。判断的方法多种多样。

对于 frame 的切换则灵活的多。

在 frame 中不能访问到主网页或其他 frame 的内容，也不能直接切换到其他 frame。

对话框也被视为子窗口。

8. 前进后退

9. Cookies

构建 cookie_dict 时可以使用以下键值对：

  1. name：cookie 的名称

  2. value：cookie 的值

  3. expire：过期时间，GMT格式，过期失效。

  4. max-age：与上一个二选一，使用整数以秒表示。

    寿命的默认值为 session，即在一个 session 内生效。

  5. domain：cookie 所属的域。

  6. path：cookie 所属路径。

  7. secure：cookie 是否使用 https 传输。

  8. httponly：必须使用 http/https 传输。

10. 等待页面加载

使用 ajax 技术的网站并不会一次返回所有的页面信息，因此我们需要等待页面加载完毕后进行操作。

最简单的是使用 time.sleep() 方法直接睡眠一段时间，但这种方法并不推荐，webdriver 提供了合适的等待方法。

webdriver 提供了两种等待 watis，即显示等待和隐式等待：

  1. 显示等待：让 webdriver 等待满足一定条件后执行。
  2. 隐式等待：让 webdriver 等待一定事件后查找某元素。

WebDriverWait 默认每 500ms 调用一次 EC（ExceptedCondition）方法，该系列方法执行成功后返回 true 或者不为 null 的值。

EC 中内置了许多方法可以用于判断某特定条件是否出现。

每次与元素互动都会等待指定时长。
