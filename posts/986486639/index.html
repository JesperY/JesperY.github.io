Spring-web的基础知识

发表于2023-02-14|更新于2023-02-18

在 Java 语言中，web 框架都是基于 Javaweb 三大组件的。

因此我们使用 Spring 来构建 web 工程，实际上就是使用 Spring 进行 Servlet 管理。

对于 web 工程，JavaEE 有三层架构的推荐规范，即表示层（Web 层），业务层（Service 层），持久层（Dao 层）。

而表示层又采用 MVC 设计模式，即 Model，View，Controller。

我们需要将 service 层注入 web 层的 Controller，将 Dao 层注入 Service 层。

关于 JavaEE 三层结构和 MVC 设计模式，另开文章讨论。

为了提高效率，同时避免 Spring 容器的重复创建，通常我们使用 ServletContextListener 来创建 Spring 容器，并将其存储到 ServletContext 域中。但其实 Spring 已经为我们封装好了这样的组件，来实现 Spring 和 web 的整合，这就是 spring-web。

1. 使用 Spring-web

导入 org.springframework.spring-web 包，在 web.xml 中配置。

Tomcat 启动时创建 ServletContext，我们配置的 ConetextLoad erListener 监听到 ServletContext 的创建，自动调用其方法，从 ServletContext 中获取我们配置的初始化参数 contextConfigLocation，根据该参数指定的路径加载 applicationContext.xml 文件，至此就完成了 xml 加载 Spring 的过程。

当我们想要使用 Spring 容器时，只需要使用 spring-web 提供的工具类，调用其静态方法 WebApplicationContextUtils.getWebApplicationContext(ServletContext sc) ，将 ServletContext 对象传入，即可获取 Spring 的上下文对象。

ServletContext 对象的生命周期贯穿整个 Tomcat 的启停，可以通过很多方法获得。

Spring 通常也会采用配置类的方式，而不是 xml 文件。

此处初始化参数的值是我们自定义的类myPackage.MyClass，该类要求继承 org.springframework.web.contex.support.AnnotationConfigWebApplicationContext ，Listener 会根据我们配置的类名创建对象，我们要在自己写的类中加载配置类。

Listener 创建 MyClass 时会调用其构造器，而构造器则调用继承的 register() 方法，该方法会负责加载配置类并创建 Spring 容器。
