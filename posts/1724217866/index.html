Spring-基于xml的bean配置

发表于2023-01-31|更新于2023-02-03

1. Spring 中读取 xml 配置文件的两种方式

Spring 可以使用 xml 配置，也可以通过注解配置，而使用 xml 配置有如下两种读取配置文件的方式。

1.1 BeanFactory 读取 xml 文件
1.2 ApplicationContext 读取 xml 文件
1.3 两者关系
  • BeanFactory 是 Spring 早期接口。而 ApplicationContext 是新版 Spring 对 BeanFactory 的封装，简化了使用方法并提供了更多更强大的功能。
  • BeanFactory 在加载配置文件时并不会加载 Bean，只在第一次使用 getBean() 时才创建要获取的 Bean 对象。而 ApplicationContext 在加载配置文件的同时会创建 Bean 对象。
  • ApplicationContext 底层仍然是使用 BeanFactory 来实现的。

我们添加了一个 bean 标签，每一个 bean 标签指示 Spring 添加一个 Bean，该标签有许多属性，接下来我们将一一讲解各个属性的含义

  • id="Bean1"：该属性指明当前 Bean 的 id，在加载时会被转换为 bean 的别名，我们在使用 getBean("xxx")时使用别名来获取 bean 对象。如果不指定 id 属性，同时也不指定其他别名，仅保留 class 属性，Spring将会把 class 属性指定的全限定类名作为获取 bean 的“别名”。

    beanFactory 在底层维护一个名为 singletomObjects 的 Map，以别名为 key，对应的 bean 对象为值。即通过 key 来获取对应的 bean 对象。

  • name="myBean,RubySIU_Bean"：如果没有指定 id 属性，则该属性指明当前 bean 的别名，多个别名使用逗号","隔开，可以在 getBean("xxx")使用该别名来获取 bean。

    没有指定 id 时，name 属性指定的第一个值将作为 bean 的别名，存储在 Map 的 key 中。后续其他的 name 值使用一个名为 aliasMap 的 Map 映射到别名。

xml 中不允许同 id 或同 name，但当两者都不指定时，可以存在多个相同 class 属性的 bean，此时底层将 bean 命名为 “<class>#0”、”<class>#1”，<class>指 class 属性的值。此时仍然可以通过别名获取到对应的 bean，如果只是用全限定类名而不加 #1 后缀，将默认获取 #0 对象。但这种配置方法毫无意义，仅作了解。

  • class="one.rubysiu.spring.Bean"：此属性指定 bean 的全限定类名，用于底层通过反射找到目标 bean 类对象，从而获取其控制权。
  • scope="prototype"：指定 bean 的作用范围，有 prototype 和 singleton 两个取值，默认为 singleton 单例模式，即创建 bean 对象后，将其存到单例池中，仅维护该单个 bean 对象，所有引用都是同一个的对象。prototype 原型模式表示每次获取 bean 对象时都创建一个新的对象，可认为每次都 new 一个新的对象。

高级 Spring 环境中（例如 spring-webmvc 组件），scope 属性的取值会有更多的选择

  • lazy-init="true"：该属性指定是否懒加载，设定为 true 后 ApplicationContext 会在调用 getBean() 时才加载 bean，而不是默认的容器创建时（加载配置文件时）加载。此属性对上一节中的 BeanFactory 加载方式无效。

  • init-methon="init"：指定 bean 对象的初始化方法，其值为方法名且要求该方法为无参方法。该方法在创建对象后由 Spring 自动调用进行用户自定义的初始化操作。

    bean 对象可以通过实现 initializingBean 接口继承一个 afterPropertiesSet() 方法，该方法的调用时机早于 init-method 属性。从方法名就可以看出，该方法晚于 Spring 的依赖注入（依赖注入相关内容暂不讨论）。

  • destroy-methon="destroy"：同上，但是该方法只有在 Spring 容器显式调用关闭方法时才会执行。

3. bean 的实例化-构造方法

Spring 在创建 bean 对象时默认调用其无参构造函数，但当 bean 没有无参构造函数时，或者我们需要调用有参构造函数时，需在 xml 文件中进行配置。

consturctor-arg 标签指定构造参数，其 name 属性指定形参名，value 指定要传递的值。对于基本数据类型或字符串类型，其值用引号括起来，由 Spring 自动进行类型转换。而对于引用类型参数，则有更低层次的标签进行指定，例如 list 或者 map 标签，因为该标签写法非常少见且较为繁琐，其详细写法请自行查阅。

5. Bean 的注入

此即为通常所说的依赖注入，即当我们在一个 Bean 中想要调用另一个 Bean 对象时，不是采用自己 new 一个新对象，而是让 Spring 容器替我们将对象创建好并传递进来的方式。

xml 配置的依赖注入有两种方式，一种是通过 set 方法注入，另一种是通过构造方法注入。

依赖注入要求被注入的 bean 中必须要有对应的属性以及 get 方法。

以上展示了多种依赖注入的方法，其中 constructor-arg 与 property 标签的用法雷同，类比使用即可。

constructor-arg 标签进行构造方法注入，即创建对象时调用对应的构造方法传递指，而 property 标签进行 set 方法注入，将 property 标签的 name 属性首字母大写与 “set” 拼接后转变为 set 方法的方法名，找到对应的 set 方法后调用该方法进行传值。

<property name=”property1” ref=”Bean1”></property> 中，name 的值将被拼接为 “setProperty1” 并找到同名方法 setProperty1(Bean1 bean1) 来进行传值。

  • 对于基本数据类型，使用 name 指定形参名，value 指定值。
  • 对于引用数据类型，使用 ref 引用其他 bean 的别名，或者直接使用 bean 标签作为值，亦或有其他写法，符合规范即可。
  • 对于集合类型，使用对应的集合子标签，在子标签内指定值。

此处写法并不唯一，选择自己习惯的或者遵循规范即可，更具体的写法规范请自行查阅。

6. 自动装配

上一节所讲的方法为 bean 的手动装配，Spring 同时提供了自动装配的方式，即不使用 property 标签，而是根据指定的匹配方法自动进行依赖注入。

bean 标签中有一个 autowire 属性，可设置为 byName，即当前 bean 中有 set 方法，例如 setBean3(Bean3 bean)，Spring 会去除 “set”，寻找别名为 Bean3 的 bean 自动调用该 set 方法进行注入。autowire 也可以取值为 byType，指明按照类型进行匹配注入。

byName 要求不存在同名 bean，byType 要求不存在同类型 bean。

使用自动装配可以省略 property 标签。

7. 一些额外的配置

此处为额外内容，知识不嫌多。

beans 标签除了作为 xml 文件的根标签之外，还有一个 profile 属性用于指定不同的开发环境。不同的开发环境之间的 bean 配置不相通，最顶层的根 beans 标签代表默认环境，我们默认使用此环境下的 bean 配置，若不指定我们将无法使用 dev 和 test 环境下所配置的 bean 对象。

可以通过虚拟机参数 -Dspring.profiles.acitve=test 或者在程序中指明 System.setProperty(“spring.profiles.active”,”test”) 来指明环境。

import 标签用于导入其他配置文件，可以将各种不同的配置分类别管理，然后 import 引入主配置文件一同加载。

alias 标签用于指定额外别名，将 name 指定的 bean 添加一个 alias 指定的别名，该标签等同于在 bean 标签中使用 name 属性指定的多个别名。
