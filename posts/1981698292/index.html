Servlet 基础知识

发表于2023-02-07|更新于2023-02-20

1. 什么是 Servlet

所谓 Servlet，全称为 Java Server Applet，即 Java 服务器小程序。意思是由 Java 编写的运行在服务端提供服务的一个小程序。Servlet 也是一种 JavaEE 规范，所谓规范即是接口，Java使用名为 Servlet 的接口来约束对象的行为，实现了这个接口的对象就是一个 Servlet，可以对外提供服务。

Servlet 接口定义的方法如下：

通过实现 serviec() 方法可以对浏览器请求做出响应。

但我们实际使用时并不会直接实现 Servlet 接口，因为 Java 给我们提供了更好的实现类，我们只需要继承即可。

通常在 Web 项目中，我们会选择继承 HttpServlet 类，该类对 Servlet 接口进行了实现，并添加了一些针对 Http 协议的方法，同时覆盖了 service() 方法，在方法中判断请求类型并分发给 doGet() 和 doPost() 方法，因此我们只需要重写这两个方法即可。

继承 HttpServlet 时并不建议重写 service() 方法。

我们编写好自己的 Servlet 方法后，通过配置文件告诉 Web 容器该 Servlet 对应哪个访问路径，当 Web 容器接收到对应路径的访问时，会将访问请求转发给对应的 Servlet 并调用其 service() 方法，传入请求的相关信息，service() 处理后做出响应，一次 HTTP 请求就完成了。

URL的格式：协议://domain:port/project/page

URL格式中，端口号后面指定访问的 Web 工程名，因为一个 Web 容器中可以部署多个 Web 工程，工程名后面是要访问的资源，可能是一个 html 或者 jsp 页面，可能是一张图片或者其他资源，也可能是 Servlet 访问路径。

2. Servlet 生命周期

Servlet 由 Web 容器管理，分为四个阶段：

  1. 加载和实例化：Servlet 第一次被访问时，由容器创建对象。可以通过 loadOnStartup 属性配置提前创建对象。
  2. 初始化：Servlet 实例化后，调用 init() 方法初始化，该方法仅调用一次。
  3. 请求处理：每次接收到请求时，调用 service() 方法进行处理。
  4. 服务终止：释放内存或者容器关闭时，调用该方法释放资源，然后释放该 Servlet 实例，完成垃圾回收。

3. URLPattern 配置

URLPattern 是 Servlet 的访问路径，指定该 Servlet 处理来自哪个路劲的访问请求，一个 Servlet 可以对应多个访问路径。

URLPattern有以下几种配置规则：

  1. 精确匹配：请求路径与 URLPattern 完全一致

    某 Servlet 的 URLPattern 是/page/action，则其对应的访问路径为 http://ip:port/project/page/action

    Web 项目中，浏览器将 / 识别为站点路径，即 http://ip:port/

    当页面中的请求方法所请求的路径为 page/action，无斜杠开头时，意为采用相对路径，即将所请求的路径加到当前 url 末尾作为完整路径。

    服务器将 / 识别为项目路径，即 http://ip:port/project/

  2. 目录匹配：路径中带有通配符 *

    /page/* 对 page 目录下的所有请求都由该 Servlet 处理

    如果访问路径同时满足目录匹配和精确匹配，则优先使用精确匹配

  3. 扩展名匹配：

    *.do 对扩展名为 do 的资源的访问都由该 Servlet 处理。

    该匹配不能以 / 开头

  4. 任意匹配：

    / ，当以上所有 Servlet 都无法匹配时，采用该路径指定的 Servlet。、

    /* 是一种目录匹配，这种写法会覆盖扩展名匹配，如非必要，请勿采用。

以上写法不能混写，即 /page/*/*.do 这种混用多种匹配写法是非法的。

Tomcat 中提供了一个 Default Servlet，该 Servlet 匹配 / 路径，因此当我们指定自己的 / 对应的 Servlet 时，会覆盖 Default Servlet。

自定义的 / 的配置会覆盖调静态资源的访问，这是因为对于静态资源的访问本由 Defatult Servlet 处理。

4. Servlet 的 xml 配置

我们编写完 Servlet 后需要配置 web.xml 文档从而让 Servlet 容器能够识别到我们的 Servlet。

5. Request 和 Response

Servlet 的 service() 方法接受 ServletRequest 和 ServletResponse 类型的参数，这两个对象都由容器产生，传递给 Servlet。

Java 仅提供 Servlet 规范，对请求与响应的解析都由容器完成，因此 Request 和 Response 的具体实现也是由容器负责。

5.1 Request

Request 封装了用户的请求信息。
