Spring-基于注解的配置

发表于2023-02-09|更新于2023-02-14

1. Spring 的基本注解

在 Spring 3.0 版本，进入全注解时代，该版本下 Spring 的配置均可由注解配置。

需要在 xml 文件中配置开启注解扫描

以下是 bean 标签中各属性对应的注解

注解配置时如果不指定 Bean 的别名，则会采用类名作为别名，因此即使不指定别名，也不存在多个 Bean 同名的情况，也就不会采用 xml 配置中在全限定名后加 #0，#1 来区分同类型 Bean 的办法.

若不同包下的相同类名的类被指定为 @Component，则在初始化过程中将会报错，因为使用类名而不是全限定类名作为别名，会出现 BeanName 冲突。

@Component 有三个衍生注解，用于分层开发时使用，提高其语义性。

2. 依赖注入相关注解

xml 配置中使用 property 标签注入。

注解配置中通过对需要注入的属性添加注解来实现。

注解注入并不依赖于 set 方法，即是 set 方法名不匹配，或者方法的参数类型是集合，Spring 均可以正常处理。

按类型注入到集合中，类型匹配的多个类都会被注入。

3. 非自定义 Bean 的注解配置

使用 @Bean 的方法会自动被 Spring 调用且只会调用一次，这就要求方法所在的类必须被 Spring 管理。

6. 注解整合第三方框架

通常使用 @Import 来引入第三方框架的配置类。

可以导入以下三种类：

  1. 普通的配置类

  2. 实现 ImportSelector 接口

    该接口提供的方法返回一个字符串数组，值为全限定类名，Spring 会把返回的所有类名注册到容器。

    该方法的参数是一个注解元信息对象，封装了使用 @Import 注解的类的其他注解信息，以供被 Import 的类使用。

  3. 实现 ImportBeanDefinitionRegistrar 接口

    该接口内部提供了默认实现，其中的方法向 BeanDefitinoMap 注册 BeanDifinition 对象。

后两种方法常用于框架开发。

被 @Import 的类会被自动加入 Spring 容器。

实现了后两个接口的方法会被 Spring 自动调用。
